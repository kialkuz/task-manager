# Описание
Task Manager API — backend-сервис для управления задачами с поддержкой повторяющихся задач и автоматических правил обработки.
Проект демонстрирует построение production-подобного Go backend с JWT-аутентификацией, PostgreSQL и контейнеризацией.

## Проект включает в себя следующие ключевые особенности:
    • Использована слоистая архитектура: handler (HTTP слой), service (бизнес-логика), repository (работа с БД)
    • Работа с PostgreSQL
    • Реализованы миграции (up/down) для управления структурой БД
    • Аутентификация с использованием JWT через middleware
    • Docker и Makefile: проект полностью контейнеризован с использованием Docker и Makefile для упрощения сборки и развертывания.
## Функциональность
    • Управление задачами: 
        ◦ Создание, редактирование, удаление задач. 
        ◦ Управление статусами задач (открытие, закрытие). 
        ◦ Применение правил для автоматического изменения параметров задач. 
    • Реализован механизм повторяющихся задач: 
        ◦ Поддержка расписаний (дни недели / месяц / год). 
        ◦ Автоматическое изменение параметров задач при выполнении условий.
    • API для запуска обработки задач: возможность запускать обработку задач через HTTP эндпоинт.

## Примеры эндпоинтов

POST /api/task — создать задачу  
GET /api/tasks — получить список задач  
GET /api/task?id={id} - получить задачу по id  
PUT /api/task?id={id} — обновить задачу  
DELETE /api/tasks?id={id} — удалить задачу  
GET /api/nextdate - получить следующую дату выполнения задачи  
POST /api/task/done?id={id} - отметить задачу выполненной  
GET /api/signin - авторизоваться
